# Изменения доступности полей amoCRM

Дата: 2026-04-08

## Что было изменено

В target-аккаунте `directorchinatutorru.amocrm.ru` были сняты ограничения `required_statuses` у следующих полей сделок:

- `1575217` — `Язык`
- `1575317` — `Вид уроков`
- `1576357` — `Уровень`

Для всех трёх полей через API amoCRM было установлено:

```json
{
  "required_statuses": []
}
```

Позже ограничения были возвращены, но расширены так, чтобы поля работали и в целевой воронке импорта:

- `pipeline_id = 9919562`
- `status_id = 84476154`

## Почему это пришлось сделать

Импортируемые сделки создаются в:

- `pipeline_id = 9919562`
- `status_id = 84476154`

До изменения поля были доступны только в других воронках и статусах. Из-за этого amoCRM молча не сохранял значения, даже если скрипт передавал корректный `field_id` и корректные `enum_id`.

Проверенные проблемы:

- поле `1575217` `Язык` было доступно только в `7909930`, `8117846`, `9719942`
- поле `1575317` `Вид уроков` было доступно только в `7909930`
- поле `1576357` `Уровень` было доступно только в `7909930`

## Какие статусы были привязаны раньше

До снятия ограничений у полей были такие `required_statuses`:

### `1575217` `Язык`

- `pipeline_id = 7909930`, `status_id = 66430654`
- `pipeline_id = 8117846`, `status_id = 66431438`
- `pipeline_id = 9719942`, `status_id = 79316726`

### `1575317` `Вид уроков`

- `pipeline_id = 7909930`, `status_id = 66430654`

### `1576357` `Уровень`

- `pipeline_id = 7909930`, `status_id = 81896134`

## Какие статусы привязаны сейчас

После точечной настройки поля больше не открыты для всех статусов, а ограничены прежними статусами плюс целевым статусом импорта.

### `1575217` `Язык`

- `pipeline_id = 7909930`, `status_id = 66430654`
- `pipeline_id = 8117846`, `status_id = 66431438`
- `pipeline_id = 9719942`, `status_id = 79316726`
- `pipeline_id = 9919562`, `status_id = 84476154`

### `1575317` `Вид уроков`

- `pipeline_id = 7909930`, `status_id = 66430654`
- `pipeline_id = 9919562`, `status_id = 84476154`

### `1576357` `Уровень`

- `pipeline_id = 7909930`, `status_id = 81896134`
- `pipeline_id = 9919562`, `status_id = 84476154`

Из-за этих ограничений значения не записывались в импортированные сделки, например в сделку `29867141`.

## Что изменилось в коде после этого

После снятия ограничений маппинг был возвращён на "родные" target-поля:

- `1054615 -> 1575217` (`Язык`)
- `1054625 -> 1575317` (`Вид уроков`)
- `1698261 -> 1576357` (`Уровень`)

Файлы, где это отражено:

## Результат проверки

После снятия ограничений и повторного прогона лида `30546963` в сделке `29867141` успешно сохранились:

- `1575217` `Язык` = `Хинди`
- `1575317` `Вид уроков` = `Индивидуально`

Поле `1576357` `Уровень` тоже разблокировано, но у проверочного лида значения уровня не было, поэтому в этой конкретной сделке оно не появилось.
